Document-Service API

POST /compile

Request
Header
Query Parameter
Field Default Description
format pdf The result format you want to get. Possible formats are pdf, odt, docx, doc. If none is provided, the default which is pdf will be taken.
Example call: /compile ?format=pdf
error false To make it easier to figure out the mistakes in the template, it is possible to activate embedded error. This way template errors are going to be visible in the returned document. Please note, only code errors inside the template are going to be embedded not other kind of errors. This parameter is important to prevent compile error being visible under production. The idea is to provide this flag only under development of a template, to not show the mistaken document at all under production. By providing this flag, the status code would be 200 otherwise it would be 422
Example call: /compile ?error
Body
ZIP
template.odt Template named with the correct extension, the name before the extension doesn't matter. Only one template can be included per request in a ZIP file. If you provide more than one, the last one will be taken. Other formats like docx are not available yet.
data.json JSON data you want the template to get resolved with. Here as well, the extension must be provided in the file name, the name before the extension doesn't matter. Only one file is accepted per ZIP file. If you provide more than one, the last one will be taken.
asset1 Assets that are referenced in your JSON data. Assets are usually pictures you want to embed in your document. You can include n assets into the ZIP file.
Example of how it could be referenced in JSON data:
As a string. { "imgVar" : "asset1" }
As an object. Only path must be provided. It is case insensitive. { "imgVar" :{ "contentType" : "image/png" , "path" : "asset1" , "size" : 2453 }}
As a list. The first entry that is a string and points to a file in the ZIP will be taken. { "imgVar" : [ "a" , "asset1" , "c.." ]}

Together with the three variable types, there are three ways of providing an image.
Remote by providing it as an URL. Please note, the link must be public for this option. http://...myimage.png
Local by providing the name to the asset in the ZIP. asset1
Embedded by providing the content as base64. data:image/png;base64,iV....

Response
Success The Content-Type will be the one of the format you requested.
Status Body Description
200 <<the requested document content>> When ever there is a document in the body, the response code is 200, even if it contains the error inside of the document. If you want to receive this error inside the document, you can provide the query parameter error
Error The Content-Type is always "text/plain; charset=UTF-8" .
Status Body Description
400 "any error explanation" If you didn't create the request as described.
422 "any error explanation" Error that happened during compilation. If you want to receive this error inside the document, you can provide the query parameter error
501 "not yet implemented" Happens for features that are not ready yet.
503 "any error explanation" This can happen if the service itself is unavailable or some resources inside the service.
500 "any error explanation" This happens on all other errors.

Header

template.odt You can provide the template directly in the body, no need to wrap it in a ZIP. Other formats like docx are not available yet.
Response
Success The Content-Type is always "application/json; charset=UTF-8" .
Status Body Description
200 [ "var1" , "var2" , "obj.number" ] A set with all vars that have been found in the provided template are returned as a JSON.
If you want to query a more specific set, you can provide the prefix parameter to retrieve vars starting with your prefix.

Example call: /vars?prefix=obj

You can request the raw template here too. Just add the query parameter raw like /example?raw get the ODT template

GET /extension

This method provides the binaries of the extension called Template-Assistance for LibreOffice. This extension is not necessary for writing templates. It is just an assistance, that helps you writing variables easier and drag and dropping them from your browser to the editor.
Please note, this extension is not yet production ready. It is recommended to use it for test purposes only. In any case, use it at your own risk.
Template Assistance /extension?os=linux_x86_64&app=libreoffice Linux x86_64 /extension?os=mac_x86_64&app=libreoffice Mac x86_64
/extension?os=win_x86_64&app=libreoffice Windows x86_64 /extension?os=win_x86&app=libreoffice Windows x86
